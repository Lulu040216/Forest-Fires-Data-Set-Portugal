The results from Exact F test and Asymptomatic Wald test agree each other and show that we cannot reject the null hypothesis. The Chisq statistic is about two times of F statistic, as the two has a difference of 2 degrees of freedom. Score test and LR test are both asymptomatic test, and similar to Wald test. The only difference between Wald test and Score test is the variance they use.

Heteroscedasticity

We need homoscedasticity to calculate a valid OLS SE, so we can plot a scatter plot to watch it, or use BP test as well. We found out a easier way to draw the residual plots by https://rpubs.com/sharmaar3/Residual_Plots.

plot(ols_model, which = 1)

From the plot, we are surprised to find that there is no significant heteroscedasticity. We use test to verify our observation from the plot.

bp_test_result = bptest(ols_model)

We could not reject the null hypothesis, thus we have conclude that the residuals are homoscedastic. However, we doubt the result of both tests, because we think most of our respond data are around 0, it is a serious right skewed data. And because BP test rely heavily on normality assumption, it may not work in our basic model. Though White test does not rely on normality assumption under large sample size, our model has about 200 degrees of freedom, might not enough for it to provide the correct result.

After all, at this stage, the variance of the random error terms non-constant is not a function of any of the quantitative covariates.

Jackknife

n = dim(df_quant)[1]
p = dim(df_quant)[2]

beta = ols_model$coefficients
SE_ols = summary(ols_model)$coefficients[, 2]

# Jackknife
betajack = matrix(0, n, p)
for (i in 1:n)
{
  jack = lm(area ~ ., df_quant[-i, ])
  betajack[i,] = jack$coefficients
}

SE_jack = apply(betajack, 2, sd) * sqrt((n - 1)^2 / n)

# Bootstrap
nboot = 10000
betaboot = matrix(0, nboot, p)
for (k in 1:nboot)
{
  boot = lm(area ~ ., df_quant[sample(n, replace = TRUE), ])
  betaboot[k,] = boot$coefficients
}
betabar_boot = colMeans(betaboot)
# Calculate Bootstrap SE
SE_boot = apply(betaboot, 2, sd)

Transformation

Transformation is vital for our data.

library(MASS)

bc <- boxcox(lm(I(area + 1) ~ ., data = df_quant), 
             lambda = seq(-2, 2, 0.1), plotit = TRUE)

best_lambda <- bc$x[which.max(bc$y)]

if (abs(best_lambda) < 0.1) {
  df_quant$trans_area <- log(df_quant$area + 1)
} else {
  df_quant$trans_area <- ((df_quant$area + 1)^best_lambda - 1) / best_lambda
}
print(best_lambda)

Thus, we can use the log transformation to area .

The fire weather variables DMC, DC, and ISI are positive indexes and can be highly right-skewed. The wind speed is also positive. Therefore, we use log1p() for these covariates, because it keeps the transformation stable even if a future data set contains zeros. For FFMC, most observations are close to the upper bound 100, so we use the log distance from the upper bound.

df_trans <- df

df_trans$log_area <- log1p(df_trans$area)
df_trans$log_DMC <- log1p(df_trans$DMC)
df_trans$log_DC <- log1p(df_trans$DC)
df_trans$log_ISI <- log1p(df_trans$ISI)
df_trans$log_wind <- log1p(df_trans$wind)
df_trans$log_FFMC_deficit <- log(pmax(100 - df_trans$FFMC, .Machine$double.eps))

Month is also seasonal. December and January are close in time, but a factor model does not know that. A second approach is to transform month into sine and cosine terms, which captures a smooth yearly cycle using only two parameters.

df_trans$month_num <- match(df_trans$month, tolower(month.abb))
df_trans$month_sin <- sin(2 * pi * df_trans$month_num / 12)
df_trans$month_cos <- cos(2 * pi * df_trans$month_num / 12)

The AIC-selected model keeps only month_cos, while the BIC-selected model keeps no covariates. This suggests that there is weak evidence of a seasonal pattern in burned area, but the overall explanatory power of the linear model is very limited. Therefore, the final model should be interpreted mainly as a descriptive model rather than a strong prediction model.

The weekend variable is not statistically significant in this model for burned area. This does not contradict the idea that fires may occur more often on weekends, because occurrence probability and burned area are different outcomes.
